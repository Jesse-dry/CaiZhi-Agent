# 采智（CaiZhi Agent）—— 材料学科 AI 学习智能体

> 面向材料专业学生的智能学习系统 —— 从"获得答案"到"理解因果链、讲清知识、迁移应用"。
>
> 清华大学 · 本科生作品 · AI+材料大赛参赛项目
>
> 🔗 **GitHub**: [https://github.com/Jesse-dry/CaiZhi-Agent](https://github.com/Jesse-dry/CaiZhi-Agent)

## 项目简介

本项目针对材料科学与工程专业的学习特点，基于**英文原版教材知识库**、**材料学科知识图谱**与**大语言模型（LLM）**，构建了一个集以下五大核心能力于一体的 AI 学习智能体：

| 核心能力 | 说明 |
|---|---|
| **智能答疑** | 基于教材知识库的精准问答，拒绝"幻觉"，所有回答有据可查 |
| **错题溯源诊断** | 不止判断对错，更追溯错误背后的知识薄弱点与典型迷思概念 |
| **苏格拉底式引导** | 通过层层追问，引导学生自己发现答案，培养材料科学因果推理能力 |
| **费曼学习法评价** | 让学生用"自己的话"讲解概念，AI 评估其是否真正理解而非机械记忆 |
| **个性化学习路径推荐** | 基于学生画像（知识点掌握度、错题分布、学习风格），动态推荐下一步学习内容 |

**核心理念**：大模型不应只是"答案生成器"，而应成为引导学生深入理解材料科学内在逻辑的**教学智能体**。

### 各层职责

- **前端交互层**：使用 Streamlit / Gradio 构建轻量、易用的 Web 交互界面，支持文本输入、图片上传（相图/显微组织等）、语音交互。
- **教学智能体层**：5 个专用 Agent 协同工作，由调度器根据用户输入自动路由到合适的教学模式。
- **知识增强层**：RAG 检索增强生成 + 知识图谱推理 + 术语对齐 + 错题-知识点映射 + 学生画像，构成多层知识护栏，确保生成内容的准确性与教学适配性。
- **数据层**：结构化与非结构化数据统一管理，支撑上层检索与推理。

## 五大 Agent 详解

### 1. 答疑 Agent
- 基于英文教材 RAG，提供**有引用来源**的精准回答
- 支持中英文混合提问，自动术语对齐
- 当检测到学生概念模糊时，自动衔接苏格拉底 Agent

### 2. 苏格拉底 Agent
- 不直接给答案，通过**层层反问**引导学生推理
- 典型案例："为什么淬火能提高钢的硬度？"→ "硬度与微观组织结构有什么关系？"→ "马氏体是如何形成的？"
- 适配材料科学中大量**因果链**知识（工艺 → 组织 → 性能）

### 3. 费曼 Agent
- 让学生**用自己的话讲解**一个概念或原理
- AI 从**完整性、准确性、逻辑链、通俗度**四个维度评价
- 识别"表面理解"（只会复述术语但无法解释本质）并给出反馈

### 4. 错题诊断 Agent
- 不仅仅标记对错，而是**追溯错误根因**
- 结合错因库，将错题映射到具体知识点缺陷
- 输出：错因分析 → 关联知识点 → 推荐补救练习 → 同类变式题

### 5. 图表讲解 Agent
- 针对材料学科大量图表（相图、CCT/TTT 曲线、应力-应变曲线等）
- 支持上传图片，AI 解读图中的关键信息
- 可反向出题：给定图表，请学生读出关键参数

## 快速开始

> ⚠️ 项目处于早期开发阶段，以下为规划中的启动流程。

```bash
# 1. 克隆仓库
git clone https://github.com/Jesse-dry/CaiZhi-Agent.git
cd CaiZhi-Agent

# 2. 安装依赖
pip install -r requirements.txt

# 3. 配置环境变量（API Key 等）
cp .env.example .env
# 编辑 .env 填入你的 LLM API Key

# 4. 准备数据
# 将英文教材 PDF 放入 data/textbooks/
# 确保知识图谱 data/kg.json 和术语表 data/glossary.csv 就位

# 5. 构建向量索引
python scripts/build_index.py

# 6. 启动应用
streamlit run frontend/app.py
# 或
python frontend/app.py  # Gradio 版本
```
